# Exercicio-2-de-modelagem-de-dados

Sistema Bancário – Arquitetura Multicamadas

Trabalho da disciplina de Modelagem de Dados.

Arquitetura do Sistema

O sistema foi desenvolvido utilizando Arquitetura Multicamadas, com a seguinte divisão:

View: Interface em HTML
Controller: main.py
Service: Camada de regras de negócio
Repository: Camada de acesso aos dados (simulação de banco de dados)
Model: Classes Cliente, Conta e Histórico
Fluxo da aplicação:
View → Controller → Service → Repository → Model
Funcionalidades


Para abrir a interface (View):

Abrir o arquivo:

view/index.html

em um navegador (Chrome, Edge, etc).
